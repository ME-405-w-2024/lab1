Control a Physical Thing

An efficient way to control the torque in a DC motor or brightness of an LED is pulse width modulation. Pulse width modulation uses a square wave of varying duty cycle (the fraction of time a signal is on) to modulate the average voltage. In the drawing below, a voltage of VM delivers full voltage to a motor and 0 volts delivers zero net voltage, so as the PWM duty cycle changes from about 40% to about 80%, it causes an increase in motor torque, assuming roughly constant motor speed.

Most microcontrollers contain timer/counter hardware which, when configured properly, creates square waves with controllable duty cycles. Once the hardware has been configured correctly, you just have to tell it the percentage of time the signal should be on as a fraction (or percentage) of the wave period, and the hardware will keep sending that wave until told to do otherwise.

Motor Control

Your ME405 kit includes an L6206 motor driver â€œshieldâ€, a circuit board which can be attached atop the Nucleo and used to control two DC motors. It has a dual H-Bridge chip which can control the magnitude and direction of the torque produced by each of two motors.

An H-bridge motor driver has logic inputs to control which transistors in the bridge are turned on at any given time and to ensure that the two transistors in each half-bridge are never turned on at the same time, as this would short power to ground and let lots of smoke out. An excerpt from the L6206 motor driver ICâ€™s datasheet is shown below:

One of the chipâ€™s two H-bridges consists of the four N-channel FETâ€™s in the right half of the diagram, with the motor connected to pins OUT1A and OUT2A. The microcontroller controls pins ENA, IN1A, and IN2A. A common way of using these pins is to set ENA high to enable the motor, set IN1A low, and send a PWM signal to IN2A to power the motor in one direction; reverse the signals to IN1A and IN2A to power the motor in the other direction. A truth table in the L6206 datasheet specifies the logic between these pins and what the chip does to the motor.

L6206 Pin CPU Pin Timer Ch. L6206 Pin CPU Pin Timer Ch.
ENA/OCDA PA10 - - ENB/OCDB PC1 - -
IN1A PB4 TIM3 1 IN1B PA0 TIM5 1
IN2A PB5 TIM3 2 IN2B PA1 TIM5 2

The table above shows the connections between the logic input pins on the L6206 and the CPU pins on the Nucleo motor control expansion board. The ENx and OCDx pins are connected together on the motor driver board, so it is best to set up the microcontroller pins connected to ENx/OCDx as open-drain outputs with pullup resistors enabled so that the microcontroller can read the pins to detect motor driver problems. The INnx pins can be set up as regular push-pull outputs.

    Note: Unlike LEDs, the motor drivers use regular non-inverted PWM logic, so set up your timer channels in mode pyb.Timer.PWM mode, not PWM_INVERTED mode.
